# Agent Operating Contract — Market Forge

## Source-of-truth convention (READ FIRST)
- The ONLY hand-edited spec files are `.ai/context.md` (the project contract) and
  `.ai/pipeline.md` (the generic agent-skills lifecycle). Everything else is
  GENERATED by `bun run sync:ai` (scripts/sync-ai-docs.ts) — NEVER edit a generated
  file: your edit is overwritten on the next sync, and the pre-commit hook blocks
  committing a stale or hand-edited one.
- How each entry file is produced:
  - `AGENTS.md` and `.ai/generated/rules.mdc` INLINE the full contract — readable
    everywhere (GitHub web view, Codex, any tool without `@`-import support). These are
    the canonical copies.
  - `CLAUDE.md` and `GEMINI.md` are thin `@`-import stubs — Claude Code and Gemini CLI
    resolve imports, so the contract is NOT duplicated into them.
  - `.cursor/rules/00-context.mdc` is a symlink → `.ai/generated/rules.mdc`. Add another
    IDE by symlinking its rules path to that artifact — no script change needed.
  - opencode reads `.ai/context.md` + `.ai/pipeline.md` + `.ai/memory.md` directly.
  - Commands (`.ai/commands/`), subagents (`.ai/agents/`), and skills (`.ai/skills/`) are
    hand-edited sources too; `sync:ai` fans them into every tool's format. A skill may use a
    Claude-Code-only accelerant internally (the dynamic `Workflow` tool, `/graphify`) as long
    as it degrades gracefully for the other CLIs — see the `evolve` skill for the pattern.
  - `.ai/workflows/*.workflow.js` are Claude-Code-only dynamic `Workflow` scripts. They are
    the single source of truth, NOT generated into any tool tree — the `Workflow` tool reads
    them by path. See `.ai/workflows/README.md`. Other CLIs achieve the same fan-out with
    their native sub-agents.
- All spec/planning/acceptance artifacts live under `.ai/specs/` (`00-requirements.md`,
  `01-spec.md`, `02-plan.md`, `03-review.md`, `97-evolution.md` (from `/evolve`),
  `98-nice-to-haves.md`, `99-acceptance.md`).
- Memory (`.ai/memory.md`) is LOCAL and gitignored — see the Memory protocol below.
- To change a convention: edit `.ai/context.md` / `.ai/pipeline.md`, run `bun run
  sync:ai`, and reflect it in `.ai/specs/01-spec.md`. At the end of every `/build` slice,
  confirm the sources still match reality; if they drifted, fix and re-sync in the same
  slice. A PR that changes a convention without re-running the sync (leaving generated
  docs stale) is incomplete.

## Project
Market Forge is an interactive Rust CLI + SDK that generates a tailored matching-engine
**venue** from the user's design choices (matching algorithm · book data structure ·
concurrency model · risk/liquidation/perps/prediction-market options). `market-forge new
my-venue` runs a wizard, then stamps out a self-contained Cargo workspace that compiles,
tests, and benches out of the box. It is a native re-implementation seeded by the design
lineage of `joaquinbejar/OrderBook-rs` (LMAX/DMAX Disruptor); upstream is credited in
`NOTICE.md`, not vendored. Audience: Rust engineers and quant-curious devs evaluating
matching-engine designs. **Dominant constraint: every generated workspace MUST compile,
pass its golden tests, and bench cleanly — a generator that emits broken code is worthless.**
Secondary: deterministic, fund-safe arithmetic (no `f64` for price/qty; no order/quantity
loss on the matching hot path).

## Stack (LOCKED — do not relitigate)
Rust 1.94 stable (edition 2021) · `clap` (derive) CLI + `inquire` interactive wizard ·
`tera` templates for codegen (swap parts of an existing tree, not regenerate-all) ·
`serde` + explicit boundary validators · `rust_decimal` deterministic arithmetic
(prices/quantities) · in-memory book + optional WAL/event-sourcing (NO external datastore) ·
lock-free hot path (`crossbeam`, `disruptor`) · `tracing` logger (sampled, off the hot path) ·
`cargo test` + `proptest` (property) + `loom`/`shuttle` (concurrency) + `criterion` (bench) ·
`ratatui` TUI · `axum` + TradingView Lightweight Charts web UI · `clippy` + `rustfmt`.
NO GPL/LGPL dependencies (constrains downstream commercial use of generated code) ·
NO `f64` for money · NO `unsafe` without a `// SAFETY:` justification · NO required async
runtime on the matching hot path · NO network calls in unit tests.

## Definition of Done (all must pass)
Tests green + coverage ≥ 80% on SDK/engine crates (`cargo llvm-cov`, fall back to documented
manual evidence if the tool is unavailable) · typecheck clean (`cargo check --workspace
--all-targets`) · linter/formatter clean (`cargo clippy --workspace --all-targets -- -D
warnings` && `cargo fmt --all --check`) · `cargo audit` clean (no open RUSTSEC advisories) ·
every external input validated at the boundary · no secrets in logs · new errors recorded in
`.ai/memory.md` · generated agent docs in sync (`bun run sync:ai`, enforced by the pre-commit
hook). **Project-specific gate: any change to the generator must regenerate at least one
sample venue and prove it `cargo build`s, `cargo test`s, and `cargo bench`es green.**

## Hard rules
NEVER push / open PRs / create remote branches / fork remote repos / add collaborators /
deploy / contact upstream maintainers. Commit locally; the human pushes and does any GitHub
fork + Joaquín outreach (CLA-140). NEVER commit `.env` or any secret.
Project additions: deterministic arithmetic only (never `f64` for money) · preserve upstream
attribution in `NOTICE.md`/README · no live network calls in unit tests · every generated
workspace must build+test+bench before its slice is considered done.

## Memory protocol (shared across Claude Code, Codex, Gemini, opencode)
- `.ai/memory.md` is a LOCAL, per-developer working log — it is **gitignored**, not
  committed. Seed it from the committed template `.ai/memory.example.md` (this happens
  automatically on `bun run sync:ai`).
- It is referenced (`@.ai/memory.md`), never inlined into committed files — that would
  leak local notes into git and churn the generated docs on every append.
- DURABLE, team-facing decisions do NOT go here. They go in the commit message and/or a
  short ADR under `docs/adr/` — reviewed and committed. Memory is for ephemeral,
  per-dev working notes only.
- APPEND, never rewrite. Keep entries terse, newest at the bottom of `## Log`. Shapes:
  - `### [YYYY-MM-DD] build-error` — `symptom → root cause → fix`.
  - `### [YYYY-MM-DD] gotcha` — environment/library quirk found the hard way.
- NEVER write a secret (private keys, `.env` values, signed payloads, bearer tokens) — even
  though it is local, the example is committed and the habit leaks.
